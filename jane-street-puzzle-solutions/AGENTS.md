# Repository Instructions

1. Read existing documentation and metadata before changing a puzzle.
2. Preserve repository conventions and do not delete unrelated work.
3. Do not copy Jane Street puzzle statements, images, or official solutions; write original summaries and reasoning.
4. Maintain complete, equivalent English and Simplified Chinese explanations for any solved puzzle.
5. Use canonical official URLs and update both metadata and indexes.
6. Independently verify every claimed answer; never label a result verified merely because code runs.
7. Document assumptions, uncertainty, and blockers honestly.
8. Add deterministic tests where computation is useful and run all validations before completion.
9. Work in manageable chronological batches.
10. Show test and validation results honestly.
